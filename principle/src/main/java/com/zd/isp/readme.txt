isp接口隔离原则：
        如果类的接口不是内聚的，就表示该类具有胖的接口，换句话说，类的接口可分解成多组方法，每一组方法都服务于一组不同
        的客户程序，这样，一些客户程序可以使用一组成员函数，而其他客户程序可以使用其他组的成员函数

        客户程序看到的应该是多个具有内聚接口的抽象基类。

        接口污染：
                接口被一个不需要的方法污染了，某个子类需要方法时就在基类中加入方法，这会进一步污染基类接口，


        分离客户就是分离接口：
                客户对接口施加的反作用力：
                                        我们往往考虑接口的变化会怎样影响他的使用者，尝试迫使接口变化的是它的使用者。


        接口隔离原则：
                   不应该强迫用户依赖于他不用的方法

                   接口需要操纵同样的数据时，需要在同一个对象中实现，如何分离必须在一起实现的接口。
                   答案：一个对象的客户不是必须通过该对象的接口去访问，也可以通过该对象的基类去访问。
